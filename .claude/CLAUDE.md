# Claude Code 필수 행동 규칙

> 이 규칙은 반드시 따라야 하며, 위반 시 토큰 낭비가 발생한다.

---

## 즉시 적용 규칙

### 파일 탐색 요청 시
```
조건: "~찾아줘", "~어디있어", "구조 분석", "에러 추적" 요청
행동: 반드시 Task(subagent_type: "Explore") 사용
금지: Glob, Grep 직접 호출 금지
```

### 파일 읽기 시
```
조건: 100줄 이상 파일
행동: offset, limit 파라미터 필수 사용

조건: 여러 파일 필요
행동: 한 응답에서 Read 병렬 호출

조건: 이미 이 대화에서 읽은 파일
행동: 다시 읽지 않음
```

### 파일 수정 시
```
조건: 수정 요청
행동: old_string은 3-10줄 이내 최소 고유 문자열만

조건: 동일 패턴 여러 곳 수정
행동: replace_all: true 사용

조건: 여러 파일 수정
행동: 한 응답에서 Edit 병렬 호출
```

### Task 모델 선택
```
조건: 단순 검색, 파일 찾기
행동: model: "haiku"

조건: 복잡한 아키텍처 설계
행동: model: "opus"

기본값: model: "sonnet"
```

### Bash 명령어
```
조건: 빌드 + 배포 요청
행동: && 체이닝으로 한 번에 실행
예시: flutter build web --release --wasm && firebase deploy --only hosting

금지: cat, grep, find, sed, awk, echo > (전용 도구 사용)
```

### 응답 작성
```
조건: 수정 완료
행동: 파일명 + 변경 핵심 1-3줄만 출력
금지: 전체 코드 출력, 장황한 설명

조건: 여러 파일 수정
행동: bullet point로 파일별 요약
```

## 금지 행동

1. **요청하지 않은 작업 수행 금지**
   - 자동 리팩토링
   - 자동 주석 추가
   - 자동 import 정리
   - 자동 코드 스타일 변경

2. **불필요한 확인 질문 금지**
   - "정말로 수정할까요?"
   - "이렇게 해도 될까요?"
   - 명확한 요청은 바로 수행

3. **중복 작업 금지**
   - 읽은 파일 다시 읽기
   - 설명한 내용 재설명
   - 같은 검색 반복

## 자동 행동

### 사용자가 파일 경로 없이 요청 시
```
1. Explore agent로 파일 찾기 (model: haiku)
2. 찾은 파일 읽기
3. 수정 수행
```

### 에러 발생 시
```
1. 에러 핵심만 전달 (전체 로그 출력 금지)
2. 바로 해결 시도
3. 해결 불가 시 사용자에게 간단히 질문
```

### 대규모 수정 요청 시 (10개+ 파일)
```
1. 파일 목록 먼저 확인
2. 3-5개씩 나눠서 수정
3. 각 단계 간단히 보고
```

### 배포 요청 시
```
행동: 빌드 + 배포 한 번에 실행
출력: URL만 전달
```

## 컨텍스트 관리

### 대화 내 캐싱
```
조건: 이전에 읽은 파일 내용 필요
행동: 다시 읽지 않고 기억된 내용 사용

조건: 이전에 수행한 검색 결과 필요
행동: 다시 검색하지 않고 기억된 결과 사용

조건: 사용자가 "아까", "방금", "위에서" 언급
행동: 이전 컨텍스트 참조, 재질문 금지
```

### 대화가 길어질 때
```
조건: 대화 20턴 이상
행동:
- 핵심 정보만 유지 (파일 경로, 변경 사항)
- 중간 과정 상세 설명 생략
- 결과 중심 응답
```

## Background Task 활용

### run_in_background 사용 조건
```
조건: 빌드, 테스트, 대규모 코드 생성 등 오래 걸리는 작업
행동: run_in_background: true 설정

조건: 사용자가 기다리는 동안 다른 작업 가능
행동: 백그라운드 실행 후 다른 요청 처리
```

## 도구 선택 최적화

### 도구 우선순위
```
파일 찾기: Glob > Bash find
내용 검색: Grep > Bash grep
파일 읽기: Read > Bash cat
파일 수정: Edit > Bash sed
파일 생성: Write > Bash echo
```

### 복합 작업
```
조건: 검색 + 수정 요청
행동:
1. Explore agent로 검색 (한 번에)
2. 결과 파일들 병렬 Read
3. 병렬 Edit
(각 단계 최소 호출)
```

## 출력 최적화

### 코드 출력
```
금지: 수정된 파일 전체 출력
허용: 변경된 부분 3-5줄만 (필요시)
권장: 변경 설명만 텍스트로
```

### 목록 출력
```
조건: 파일 목록, 검색 결과
행동: 최대 10개만 출력, 나머지는 "외 N개" 표시
```

### 에러 출력
```
금지: 전체 스택 트레이스
허용: 핵심 에러 메시지 1-2줄
```

## 토큰 절약 우선순위

1. **Explore agent 사용** (최우선 - 50-80% 절감)
2. **컨텍스트 캐싱** (재읽기 방지 - 30-50% 절감)
3. **파일 부분 읽기** (offset/limit - 30-70% 절감)
4. **병렬 도구 호출** (왕복 감소 - 20-40% 절감)
5. **응답 간결화** (출력 최소화 - 20-40% 절감)
6. **명령어 체이닝** (Bash 최적화 - 30-50% 절감)
7. **Haiku 모델 사용** (단순 작업 - 60-80% 절감)
